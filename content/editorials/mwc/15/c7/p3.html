Blue and Green Editorial mwc15c7p3

aurpine

This is a bitmask dp problem.

All the operations are implemented as follows.

Set

Since the marble colour would never be set to itself (that's a waste of currency), just change the colour of a certain marble. Use the xor operation for the desired bit.

Rotate

Rotate left and right are very similar. Bitshifts are used to move all the bits, but the end bits require special concern.

For a right rotate, the rightmost marble will get truncated in a bitshift. So there's no need to worry about disposing of that bit. It will have to be moved to the left.

For a right rotate, the rightmost marble will get truncated in a bitshift. Then move that bit to the far left.

The left rotate is more complex. The leftmost bit still exists after shifting. A mask has to be applied to get rid of that bit. Modding by \(2^N\) also works. The leftmost bit has to be moved to the right in the same manner as for the rightmost bit of the right rotate.

Invert All

Inverting all the marbles requires the use of xor. xor by \(2^N - 1\).

Time Complexity

\(\mathcal O(2^N \times M)\), however, this worst case barely occurs because not all the possible combinations will be reached and with all the \(M\) currencies remaining.
