Queue

The Queue is a FIFO (first in first out) data structure. It allows you to insert and remove elements in the order in which they were inserted. If an element was inserted before another element, then the first element would be removed before the second.

The following is a diagram of a queue. It is depicted as a linear structure. The first element is called the front. It is the next element to be removed. Conversely, the last element is called the back. It is the most recently inserted element.


Elements may only be inserted to the back and removed from the front.

Features

Some of the following features have two methods that support it. It is recommended to use the one that prevents thrown exceptions, however, this scenario rarely occurs, so either method may be used. The unsafe may be preferable due to their easy-to-remember names. E is the element template class.

Size

int size() – returns the number of elements in the queue.

Alternatively, boolean isEmpty() can be used if the number of elements isn't relevant and only if it's empty or not.

Insert Back

boolean add(E e) – the element in parameters is added to the queue.

Access Front

E peek() – returns the front element

Remove Front

E remove() – returns and removes the front element

Usage

Notice in the code below that the Queue is an interface and that the class is ArrayDeque. ArrayDeque is a class that supports the queue operations. Alternatively, a LinkedList could be used. A linked list is a common implementation of a queue. However, as a queue, ArrayDeque outperforms LinkedList. 


    Queue<Integer> q = new ArrayDeque<Integer>();
    
    for(int n = 1; n <= 5; n++) q.add(n);
    // q now contains 1 2 3 4 5
    
    if(q.peek() == 1 && q.size() == 5) {
        while(!q.isEmpty()) System.out.print(q.remove() + " ");
    }

Output:

1 2 3 4 5

Applications

Queues aren't data structures that you may find applicable for most problems. However, they are fundamental in some other algorithms, especially in the field of graph theory.
