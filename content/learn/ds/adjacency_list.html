Adjacency List

An adjacency list is used to store edges of a graph. It is often used for sparse graphs and/or graphs with multiple edges going between the same pair of nodes.

Declaration

Adjacency lists are used to represent edges between nodes in a graph. For each node in the graph, the adjacency list has a 'list' containing the edges that connect the node to other nodes in the graph.

If the graph is unweighted, the declaration of the adjacency list might look like this:


    // initialize an array of ArrayLists, one for each of the N nodes in the graph.
    List <Integer> [] adj = new ArrayList [N];

The values contained in the ArrayList adj [v] represent the other nodes that vertex \(v\) is connected to. For example, if adj [2] contains the values \(\{1, 3, 5, 3\}\), this means that edges currently exist between vertex \(2\) and vertices \(\{1,3,5\}\). Note that because vertex \(3\) appears twice in the list, there are two different edges going between vertex \(2\) and \(3\).

For weighted graphs, it is neccessary to make the adjacency list store the vertices as well as the weights of the edges. To do this, an Edge class can be created, storing the two vertices connected to each other as well as the edge weight. The adjacency list will now consist of ArrayLists which store the edges. This is illustrated as below.


    // initialize an array of ArrayLists, one for each of the N nodes in the graph.
    List <Edge> [] adj = new ArrayList [N];
    
    // an Edge class, where each edge stores the two vertices connected to each other as well as the weight of the edge connecting them.
    class Edge {
      int start, end, weight;
      
      public Edge (int start, int end, int weight) {
        this.start = start; this.end = end; this.weight = weight;
      }
    }

Adding an Edge

Adding edges is done differently based on the graph.

Weighted Undirected

This is the most common type of graph. Given a b w, where a and b are the two nodes to connect and w is the edge weight.


    adj [a].add (new Edge (a, b, w));
    adj [b].add (new Edge (b, a, w));

Weighted Directed

Given a b w, meaning a one way edge from a to b with a weight of w.


    adj [a].add (new Edge (a, b, w));

Unweighted Undirected

Given a b, a two way edge between a and b.


    adj [a].add (b);
    adj [b].add (a);

Unweighted Directed

Given a b, a one way edge between a and b.


    adj [a].add (b);

Space Complexity

\(\mathcal O (V + E)\), where \(V\) and \(E\) represent the number of vertices and edges in the graph.

Time Complexity

Adding edge

\(\mathcal O (1)\) amortized, as the adding of an edge could trigger the resizing of the ArrayList which takes \(\mathcal O (N)\). Happens rarely though.

Checking If Edge Exists

\(\mathcal O (E)\), where \(E\) is the number of edges connecting the vertex.

Iterating Adjacent

\(\mathcal O (E)\), where \(E\) is the number of edges connecting the vertex.

Get all Connected Vertices

\(\mathcal O (1)\)
