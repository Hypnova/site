HashMaps

A HashMap is a class that utilizes pairs. Similar to a relation in mathematics, a key is associated with a corresponding value.

HashMaps are unordered collections, meaning when iterating through them, insertion order is not necessarily maintained.

Declaration and Implementation

HashMaps generally have two identifiers, a class type for the key, as well as a class type for the value. A simple declaration of a HashMap involving the association of an integer with a string would look like: 


    // initialize an array of ArrayLists, one for each of the N nodes in the graph.
    HashMap <Integer><String> map = new HashMap<Integer><String>();

After initializing the HashMap, we can begin to store &LtKey,Value&qt pairs inside of it: 


      map.put(1, "How's it going");

This pair has (1) as it's key. The value associated with this key is "How's it going"

When we use the get method, which accepts the key as its parameter, the associated value returned is "How's it going" 


      map.get(1);

Just a little warning, when using keys, they must reference the same object. Hence, when using two objects that may be have the identical data, yet are declared differently, it will be unable to recall the value associated with the original object.

Time Complexity

\(\mathcal{O}(N)\)

Space Complexity

\(\mathcal{O}(N)\)
