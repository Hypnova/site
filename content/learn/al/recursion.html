Recursion

Recursion is more of a general technique than an algorithm. It is where a method calls itself. This is useful for many reasons. One is that it simulates a loop, but with more complex variables and increments. It also makes it very easy to code functions in terms of subproblems or divide and conquer.

Here is a basic example of an infinite loop recursion method. 


    static void inf() {
        inf();
    }

Since the program never stops calling itself, it goes on forever.

This method counts down, using recursion. 


    static void count(int num) {
        if(num == 1) {
            System.out.println(1);
            return;
        }
        System.out.print(num + " ");
        count(num - 1);
    }

Calling count(5); will result in the following output: 

5 4 3 2 1

The method can easily be changed to make the method count up instead. 


    static void count(int num) {
        if(num == 1) {
            System.out.print(1);
            return;
        }
        count(num - 1);
        System.out.print(num + " ");
    }
  
1 2 3 4 5

Another classical recursive method is the fibonacci function. This function finds the \(n\)th fibonacci number, starting from \(1\).

$$ fib(n) = \begin{cases} 1 & \quad \text{if } 1 \le n \le 2 \\ fib(n - 1) + fib(n - 2) & \quad \text{if } n \gt 2 \\ \end{cases} $$

The recursive function is defined with it's base cases which are the terminating scenarios of a method which are required to end the recursion. In the two methods above, \(\text{num} = 1\) were the base cases. It is very easy to convert the recursive fibonacci recursion to code. 


    static int fib(int n) {
        if(n <= 2) return 1;
        return fib(n - 1) + fib(n - 2);
    }

Another fundamental rule of recursion is that the method does not call itself with the same parameter, thus causing infinite loops. Similarly, there are other ways to achieve an infinite loop. 


    static void inf(int n) {
        if(n == 0) return;
        if(n % 2 == 0) inf(n - 1);
        inf(n + 1);
    }

Calling any values of n other than -1 and 0 will result in an infinite loop. Infinite loops in recursion should not occur when done properly. Note that recursion infinite loops don't last forever. A stack overflow occurs very quickly. A StackOverflowError occurs when there are too many concurrent recursion calls thus running out of memory for the stack (where the stack variables are stored).

Recursive methods may use any instance variables in addition to the parameters. It may also create any new local variables within the method itself. You may pass anything into the method, including arrays, however, you should note what type of value passed (pass by reference/value).
