Stack

The stack is a LIFO (last in first out) data structure. It allows you to insert and remove elements in the reverse order of which they were inserted. The last element inserted is the next element to be removed. The recursion stack is similar to the stack in that they are both LIFO.

The following is a diagram of a stack. It is depicted as a linear structure. The last element inserted is called the top.


Elements may only be inserted into and removed from the top.

Features

An implementation of a stack is provided in most programming languages. In Java, this data structure is defined by both the Deque (stands for double ended queue) interface. While there is a Stack class in Java, it is slow and not used very often. In addition, the Deque class supports more operations (such as insertion and deletion from both the front and back), making it more useful in programming contests.

Note that although Deque supports more operations, only the standard stack operations are covered in this lesson. The other methods are covered in the Deque lesson.

Some operations are supported by multiple methods in the Deque interface. To reduce confusion, only one method is listed for each feature.

It is important to note that all operations below are performed in amortized \(\mathcal{O}(1)\) and that E is the type of object that is stored in the stack.

Size

int size() – returns the number of elements in the stack.

Alternatively, boolean isEmpty() can be used if the number of elements isn't relevant and only if it's empty or not.

Insert Front

void push(E e) – element e is added to the front of the stack. This method will throw an exception if e is null.

Access Front

E peek() – returns the element at the front of the stack. Returns null if the stack is empty.

Remove Front

E pop() - returns and removes the element at the front of the stack. Throws an exception if the stack is empty.

Usage

ArrayDeque is a class which implements the methods in the Deque class that were shown above. Although the LinkedList class also implements the above methods, it is almost always slower than an ArrayDeque and is not recommended.


    // create a stack which stores objects of type Integer
    Deque<Integer> stack = new ArrayDeque<Integer>();
    
    // stack now contains the elements 5 4 3 2 1 (in that order)
    for(int n = 1; n <= 5; n++) {
      stack.push(n);
    }
    
    if(stack.peek() == 1 && stack.size() == 5) {
        while(!stack.isEmpty()) {
          System.out.print(stack.pop() + " ");
        }
    }

Output:

5 4 3 2 1

Applications

Stacks are extremely important data structures that are used in many programming problems, especially in the field of graph theory, where they are most commonly associated with DFS algorithms. While they are rarely the only part of a solution to a problem, they are used as utility data structures in many solutions.
